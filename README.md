# Барбершоп «ЦЕХ» — веб-приложение с записью онлайн

Полноценный серверный сайт для барбершопа: публичная страница с записью
и закрытая панель администратора, где владелец видит заявки и отмечает
выполненные. Не статическая страница, а настоящее приложение с базой данных
и авторизацией.

**Стек:** Python (Flask) · PostgreSQL / SQLite · Gunicorn
**Деплой:** Render (бесплатный тариф, HTTPS, без покупки сервера)

---

## Что умеет

- Публичная страница барбершопа (услуги, мастера, контакты).
- Форма записи — заявки сохраняются в базу данных.
- Вход для персонала (`/admin/login`).
- Панель администратора: список всех заявок, отметка «выполнено».

## Безопасность

Приложение спроектировано с защитой от типовых веб-уязвимостей. Важно понимать:
абсолютно неуязвимых систем не бывает — здесь закрыты основные классы атак
из списка OWASP.

| Угроза | Как закрыта |
|---|---|
| SQL-инъекции | ORM SQLAlchemy формирует параметризованные запросы; пользовательские данные не склеиваются с SQL вручную. |
| XSS | Шаблоны Jinja2 автоматически экранируют любой вывод. |
| CSRF | Токены на каждой форме (Flask-WTF); запрос без токена отклоняется. |
| Слабое хранение паролей | Пароли хранятся как хеши (Werkzeug), не в открытом виде. |
| Перебор паролей (brute-force) | Ограничение частоты запросов на вход (Flask-Limiter). |
| Кликджекинг | Заголовок `X-Frame-Options: DENY`. |
| Прочее | `Content-Security-Policy`, `X-Content-Type-Options`, безопасные cookie (HttpOnly, SameSite, Secure на проде). |

## Запуск локально (на своём компьютере)

Нужен установленный Python 3.10+.

```bash
# 1. Перейти в папку проекта и создать виртуальное окружение
python -m venv venv

# 2. Активировать его
#    Windows:
venv\Scripts\activate
#    macOS / Linux:
source venv/bin/activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Создать файл .env (скопировать из примера) и заполнить
#    Сгенерировать SECRET_KEY:  python -c "import secrets; print(secrets.token_hex(32))"

# 5. Запустить
python app.py
```

Откройте в браузере `http://127.0.0.1:5000`.
Админка: `http://127.0.0.1:5000/admin/login`
(логин/пароль — те, что в `.env`).

Локально база — это файл SQLite, ничего отдельно настраивать не нужно.

## Деплой на Render (бесплатно, доступно из любой точки мира)

### Способ A — через Blueprint (быстрее)

1. Залейте проект в репозиторий на GitHub.
2. Зайдите на render.com, войдите через GitHub.
3. **New → Blueprint**, выберите свой репозиторий. Render прочитает
   `render.yaml` и предложит создать веб-сервис + базу.
4. Введите `ADMIN_PASSWORD` (придумайте надёжный) и подтвердите.
5. Дождитесь сборки — Render выдаст публичную ссылку вида
   `https://barbershop-tseh.onrender.com`.

### Способ B — вручную через дашборд (надёжный запасной путь)

1. Залейте проект на GitHub.
2. На Render: **New → PostgreSQL** → план Free → создать. Скопируйте
   «Internal Database URL».
3. **New → Web Service** → выберите репозиторий. Настройки:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app`
4. В разделе **Environment** добавьте переменные:
   - `SECRET_KEY` — длинная случайная строка
   - `ADMIN_USERNAME` — например `admin`
   - `ADMIN_PASSWORD` — надёжный пароль
   - `FLASK_ENV` — `production`
   - `DATABASE_URL` — вставьте URL базы из шага 2
5. **Create Web Service.** После сборки получите публичную HTTPS-ссылку.

База и первый администратор создаются автоматически при первом запуске.

### О чём знать заранее (особенности бесплатного тарифа)

- После простоя сервис «засыпает», и первый заход после паузы
  занимает около минуты — это нормально для бесплатного плана.
- Бесплатная база PostgreSQL на Render удаляется примерно через 90 дней
  после создания. Для демо-портфолио этого достаточно; для боевого
  проекта берётся платный план или внешняя база.

## Структура проекта

```
barbershop/
├── app.py             # маршруты, авторизация, заголовки безопасности
├── models.py          # модели базы (Admin, Booking)
├── forms.py           # формы с валидацией и CSRF
├── requirements.txt   # зависимости
├── render.yaml        # конфиг автодеплоя на Render
├── Procfile           # команда запуска для прод-сервера
├── .env.example       # пример переменных окружения
├── templates/         # HTML-шаблоны (Jinja2)
│   ├── base.html
│   ├── index.html     # публичная страница
│   ├── login.html     # вход в админку
│   └── admin.html     # панель заявок
└── static/
    ├── style.css
    └── main.js
```
